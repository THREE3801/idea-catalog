# 点子索引卡 · Idea Catalog —— 产品设计交接说明

> 2026-07-14 修订:采纳 Claude Design 出的"意图行"方向,视觉与核心输入交互完成一次重做。原始设计交接包见 `Three380 网站优化设计.zip`。

## 一句话定位
一个**单用户**的个人点子记录器。核心信条:**记录要快到没有摩擦,整理是可选的,不整理也完全可用。**

目前是单用户(现阶段唯一用户是产品主本人),无需账号/登录负担——但这是**起点,不是终局**。

## 核心设计理念(重构时不可动摇的几条)
1. **零摩擦捕获优先**:界面只有**一行输入**。用户写一句自然语言,系统实时解析出标题、截止日期、标签、链接、状态,回车归档。任何改动都不能让这一步变复杂或多一步。
2. **让界面读懂你,而不是让你填格子**:结构由解析器从自然语言里抽取,不需要点日期选择器、不需要手动选状态字段。
3. **整理是可选项,不是义务**:备注、链接、截止时间都可以事后通过"补充"按钮就地展开编辑,不整理也完全可用。
4. **防丢是底线**:数据曾经丢过一次。所以有一键导出(JSON/纯文本)和删除二次确认。
5. **安静、不喧宾夺主**:暖纸感、克制、像文具而非 SaaS 仪表盘。

## 目标用户与场景
- 现阶段唯一用户就是产品主本人,无需账号/登录/多用户。
- 典型场景:在和 AI 聊天、刷信息时冒出的选题/产品想法,用一句自然语言随手甩进来,之后再回来筛选验证。

## 已实现的功能(交互清单)
| 功能 | 行为 |
|---|---|
| 意图行捕获 | 唯一常驻输入行,自然语言,输入时实时解析预览,回车归档 |
| 自然语言解析 | 从一句话里抽取标题、链接、`#标签`、相对/绝对截止日期、状态关键词(见下方解析规则) |
| 点子行 | 显示状态圆点、标题、备注、状态文字、截止提示、标签、链接 |
| 状态流转 | 待验证 → 进行中 → 已完成 → 已放弃,点状态圆点循环切换 |
| 截止提醒 | 未完成/未放弃的点子,剩 ≤3 天或已超期显示红色紧急标签;已完成/已放弃只显示中性的"截止 YYYY/M/D" |
| 补充编辑 | "补充"按钮就地展开,编辑备注/链接/截止(标题与标签不支持事后编辑,重新记一条即可) |
| 筛选 | 按状态筛选,每个 tab 带实时计数 |
| 数据导出 | 一键导出全部为 JSON 或纯文本下载 |
| 删除确认 | 点"删"→"确认删"/"取消" 二次确认,防误删 |
| 访问口令(可选) | 右上 ⌸ 图标,轻量口令保护,配合 Supabase RLS 策略生效 |

### 自然语言解析规则
输入文本按以下顺序抽取,抽出的片段从标题里移除,剩余即标题(移植自设计稿,实现见 `src/lib/parseIntent.js`):
1. **链接**:`https?://…` 或 `www.xxx.yyy/…` 或裸域名;无协议则补 `https://`。
2. **标签**:`#词`(可多个)。
3. **截止日期**(相对"今天"计算,输出 ISO `YYYY-MM-DD`):`今天`/`明天`/`后天`/`大后天`、`N天后`/`N天内`、`[下[下]]周/星期/礼拜 + 一二三四五六日`、`M月D日/号`、`M/D`。
4. **状态关键词**:`做完了/完成/搞定/done/发布了`→已完成;`在做/进行中/正在/开工`→进行中;`放弃/不做了/算了/废弃`→已放弃;都没有→待验证(默认)。
5. **标题**:剩余文本,压缩空白并 trim;为空则回退为原始输入。

## 现有视觉规格(重构基线)

**配色**
- 页面背景:`radial-gradient(120% 90% at 50% 0%, #DAD8CE 0%, #CFCDC2 100%)`(暖纸感渐变)
- 卡片容器:`#EFEDE4`,边框 `#CFCDC2`;输入/子表面 `#FCFBF6`;编辑输入框 `#FFFEFA`
- 分隔线 `#E4E2D8`;通用浅边框 `#DEDDD3` / `#E7E5DB`
- 主文字 `#26261F`;正文/备注 `#5C5B54`;次要 `#8A8981`/`#9A998F`;更淡 `#AEACA1`/`#B0AEA3`/`#BDBBB0`/`#C6C4B9`
- 深色实心(保存按钮等强调):`#2A2A28`,其上文字 `#F4F3EE`
- 链接 `#2D5A6B`;hover `#B3423A`;金褐前缀图标 `#C9B27A`;标签 chip 底 `#EDEBE2`
- 紧急/超期红:`#B3423A`,红底 `#FBEAE8`
- 四状态语义色(前景 fg / 浅底 bg / 圆点 dot):
  - 待验证 pending:`#A9761D` / `#F6EDD6` / `#C08A2E`
  - 进行中 doing:`#2D5A6B` / `#E3EDF0` / `#2D5A6B`
  - 已完成 done:`#3F7A57` / `#E6F0E9` / `#3F7A57`
  - 已放弃 dropped:`#87867E` / `#EDEDE8` / `#ABAAA1`

**字体**
- 标题/正文/输入:`Source Serif 4`(衬线,400/500/600/700)
- 元信息/标签/按钮/数字:`IBM Plex Mono`(等宽,400/500/600)
- "衬线+等宽"的搭配是这套视觉的核心,重构时别丢。

**布局**
- 单栏居中,最大宽度约 660px,圆角 8px(容器),`box-shadow: 0 10px 30px rgba(60,55,40,0.10)`。
- 结构自上而下:容器头(左 mono 小标签"IDEA CATALOG · 意图捕获",右侧口令/导出按钮 + 计数)→ 意图输入行 → 实时解析预览(仅输入非空时出现,`popIn` 动画)→ 文字型状态筛选 tab → 点子行列表(`cardIn` 动画)。
- **点子行解剖**:左侧状态圆点(20px,可点循环切换)→ 标题行(标题 + "补充"/"删"按钮)→ 备注(有则显示)→ meta 行(状态文字 + 截止 facet + 标签 chips + 链接)→ 展开时的内联编辑区(备注/链接/截止)。

## 数据结构
```
id: string          // 唯一标识(Supabase uuid)
num: number         // 显示编号,数据库自增(不对用户可见,内部排序用)
title: string        // 标题(必填,解析后剩余文本)
note: string         // 备注(可选,仅能通过"补充"编辑)
link: string          // 链接(可选,解析自输入或"补充"编辑)
deadline: string      // 截止日期 YYYY-MM-DD(可选,解析自输入或"补充"编辑)
status: string         // pending | doing | dropped | done
tags: string[]          // 标签(可选,解析自 #hashtag,暂不支持事后编辑)
createdAt: string        // ISO 时间戳
```

## 明确的边界
- ⏳ **用户注册/登录/多用户:现阶段不做**,先打磨单人自用体验;但这是阶段性取舍,不是永久排除,以后很可能会加。所以**架构和视觉都要给"登录入口、账号态、多用户数据隔离"留出扩展余地,别做成锁死单用户的设计**。
- ✅ **轻量标签系统已经实现**(`#hashtag` 解析),这是明确要保留的功能;但不要在此之上加更复杂的分类/文件夹/多级标签体系。
- ❌ 不要自动调研/AI 分析(留扩展空间,但现在不实现)
- ❌ 不要任何付费/推广元素
- ⚠️ 重构可以再换视觉皮肤,但**不能牺牲"一行输入、回车即存"的零摩擦捕获**,也不要打乱四状态的语义。
